## 📍 2. Address Management (การจัดการที่อยู่)

### **A. API สำหรับจัดการที่อยู่ (หน้า Profile)**
- `POST /api/v1/users/addresses`: เพิ่มที่อยู่ใหม่
- `PUT /api/v1/users/addresses/:addressId`: **(ใหม่)** แก้ไขที่อยู่เดิม
- `PATCH /api/v1/users/addresses/:addressId/default`: **(ใหม่)** ตั้งค่าให้เป็น "ที่อยู่หลัก"
- `DELETE /api/v1/users/addresses/:addressId`: ลบที่อยู่

`fullName`, `phone`, `address`, `subDistrict`, `district`, `province`, `postalCode`

## 🛒 3. Order Checkout Logic (ลำดับการเลือกที่อยู่)

**สำคัญ:** ระบบเปลี่ยนมาใช้แบบ **Explicit Selection** (เลือกอย่างใดอย่างหนึ่ง) เพื่อป้องกันการส่งของผิดที่:

1.  **ลำดับที่ 1 (Manual Address):** หาก Frontend ส่งออบเจกต์ `shippingAddress` มา -> **Backend จะบังคับใช้ที่อยู่นี้เท่านั้น** (หากกรอกไม่ครบ 7 ฟิลด์ ระบบจะแจ้ง Error ทันที ไม่มีการแอบไปดึงที่อยู่เก่ามาใช้)
2.  **ลำดับที่ 2 (Saved ID):** หากไม่มี Manual แต่ส่ง `addressId` มา -> **Backend จะดึงข้อมูลจากโปรไฟล์ตาม ID นั้น** (หากหาไม่เจอจะแจ้ง Error)
3.  **ลำดับที่ 3 (Auto-Default):** หากไม่ส่งฟิลด์ที่อยู่มาเลย -> **Backend จะดึง "ที่อยู่หลัก" (Default) หรืออันแรกสุดในโปรไฟล์มาใช้ให้อัตโนมัติ** (หากผู้ใช้ไม่เคยมีที่อยู่เลย จะแจ้ง Error ให้เพิ่มที่อยู่ก่อน)
